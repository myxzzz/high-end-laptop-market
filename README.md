# 💻 算力影子：高端轻薄本硬件溢价分析分析 (数据分析实战)

> 💡 **项目协作方式（AI Driven）**：作为一个数据分析学习者，我承担了 100% 的业务痛点挖掘、影子价格漏斗设计、HHI 与 NLP 定权指标构建，但在 Python / Pandas 等底层代码编写环节，由我调度引导 AI（如 Copilot/OpenCode）辅助完成功能落地。

---

## 🔍 1. 为什么做这个项目？(业务实战背景)

在买电脑时，把 16G 内存升级到 32G 往往要多花 1500 块，可市场上单买一根 16G 内存条只要 300 块。这种利用“整机模具”把便宜的硬件卖出天价的现象，在行业里被调侃为“黄金内存/硬盘”。

**本项目试图解决的核心问题**：利用电商真实销售与配置数据，通过量化计算拆解主流品牌电脑的“硬件溢价倍数”，将那些被打包在高端轻薄本里的“黑箱利润”透明化。

## 📊 2. 核心成果展示说明

受限于赛事保密要求，真实商业结论的图表已在报告书中呈现，本开源仓库主要作技术路径展示。核心结论方向如下：

1. **市场分层与占位 (K-Means 聚类)**：成功对市场上百款复杂的机型进行了降维定组，直观展示四大价值梯队的分布。
2. **硬件溢价测算**：基于影子价格测算出了单 GB 配置的溢价倍率，量化揭示了不同品牌在存储组合上的特定溢价策略。
3. **NLP 痛点挖掘**：通过8倍定权法，成功从评论数据中提取出常规 HHI 指数无法捕捉的用户真实诉求。

## 🧠 3. 核心分析方法与技术路径

我搭建了一个从原始数据到可视化报告的分析流水线 (`scripts/etl/` 和 `scripts/local/`)：

1. **影子价格模型 (核心亮点) 🌟**：
   - **思路**：控制同机型、同处理器，只计算存储容量的差价。即 `(高配版价格 - 低配版价格) / 存储差量`，求出该品牌对单位存储的“影子估值”。
2. **K-Means 市场聚类 📊**：
   - 面对上百个复杂的 SKU，我用 Sklearn 运用 K-Means 算法对数据进行了无监督降维，将市场清晰划分为“移动工作站”、“发烧创作”等 4 个梯队。
3. **消费者弱信号挖掘 (NLP) 💬**：
   - 面对 23 万条杂乱的非结构化评论，用 `jieba` 分词清理后，我设计了**“8倍定权法”**，把原本被高频废话淹没的真实痛点（如“散热调教”、“AI交互”）成功提取了出来。

## 📁 3. 核心数据表与字段说明 (Data Dictionary)

为了保护商业隐私，项目开源的数据（存放在 `data/output/` 下）已统一步改写为 `Brand_A`, `Brand_B` 等代号。以下是项目中最重要的三张表：

### 📝 表1: `品牌硬件溢价对比分析.csv` (最核心的产出)

这张表直观展示了各品牌把硬件包装进电脑后，相比市场成本多赚了多少倍。

- `品牌`：脱敏后的厂商代号 (如 Brand_A)。
- `硬件项目`：指内存或硬盘。
- `厂商升级单价_每GB`：我用影子价格模型算出来的，买该品牌电脑时，每增加 1GB 内存/硬盘实际要付给厂商的钱。
- `零售市场单价_每GB`：基准线（公允价格）。
- `溢价倍率`：核心指标。值越大说明加价越狠。

### 📝 表2: `各品牌影子价格查.csv`

- `平均影子价格` / `中位数价格`：用来排除个别极端机型造成的偏差。
- `价格波动_标准差`：看出该品牌的定价体系是否稳定。

### 📝 表3: `电脑k均值.csv` (聚类结果说明表)

- `Group_prediction`：机器打上的 0、1、2、3 四个分类标签。
- `Count_型号`：该梯队下有多少款电脑（反映市场拥挤度）。
- `Avg_售价`：用来锚定该梯队的用户消费水平（如: 40000 代表高端工作站组）。

## ⚙️ 5. 目录结构与复现方式

```text
├── data/
│   ├── input/       # 源数据 (由于数据合规原因不提供)
│   └── output/      # 脱敏后的各类分析结果 (.csv)
├── scripts/
│   ├── etl/         # 🌟 数据清洗、指标计算、NLP 处理的底层代码库
│   └── local/       # 📊 指标可视化脚本库 (供代码逻辑参考)
├── reports/         # 生产报告相关素材
└── README.md        # 结构文档说明
```

> **运行说明**：环境需安装 `pandas`、`sklearn`、`jieba` 等基础库。为保护数据隐私，`data/input` 源数据未随仓库上传，且 `data/output` 已深度脱敏。因此 `scripts/local` 下的可视化脚本仅作代码逻辑参考，直接运行可能会缺少源文件而报错。
